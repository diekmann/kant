section‹Zahlenwelt Helper›
text‹Wir werden Beispiele betrachten, in denen wir Welten modellieren, in denen jeder Person eine
Zahl zugewiesen wird: \<^typ>‹person ⇒ int›.
Diese Zahl kann zum Beispiel der Besitz oder Wohlstand einer Person sein, oder das Einkommen.
Wobei Gesamtbesitz und Einkommen über einen kurzen Zeitraum recht unterschiedliche Sachen
modellieren.

Hier sind einige Hilfsfunktionen um mit \<^typ>‹person ⇒ int› allgmein zu arbeiten.›

text‹Default: Standardmäßig hat jede Person \<^term>‹0::int›:›
definition DEFAULT :: "person ⇒ int" where
  "DEFAULT ≡ λp. 0"

(*<*)
syntax
  "_ZahlenWelt"  :: "updbinds ⇒ 'a" ("(1🌐[_])")

translations
  "_ZahlenWelt ms"                     ⇌ "_Update (CONST DEFAULT) ms"

(*>*)


text‹Beispiel:›
lemma ‹(DEFAULT(Alice:=8, Bob:=3, Eve:= 5)) Bob = 3› by eval

text‹Beispiel mit fancy Syntax:›
lemma ‹🌐[Alice:=8, Bob:=3, Eve:= 5] Bob = 3› by eval

lemma ‹show_fun 🌐[Alice := 4, Carol := 4] = [(Alice, 4), (Bob, 0), (Carol, 4), (Eve, 0)]› by eval
lemma ‹show_num_fun 🌐[Alice := 4, Carol := 4] = [(Alice, 4), (Carol, 4)]› by eval

abbreviation num_fun_add_syntax ("_ '(_ += _')") where
  "f(p += n) ≡ (f(p := (f p) + n))"

abbreviation num_fun_minus_syntax ("_ '(_ -= _')") where
  "f(p -= n) ≡ (f(p := (f p) - n))"

lemma ‹(🌐[Alice:=8, Bob:=3, Eve:= 5])(Bob += 4) Bob = 7› by eval
lemma ‹(🌐[Alice:=8, Bob:=3, Eve:= 5])(Bob -= 4) Bob = -1› by eval
